# M-memory: Your System's Memory

This is where the system learns. Every session adds to it. Every review improves it.

---

## The 4 Files

| File                  | What Goes In                                              | Who Writes                                            |
| --------------------- | --------------------------------------------------------- | ----------------------------------------------------- |
| `learning-log.md`     | What worked, what didn't, patterns discovered             | Gatekeeper after every review. You after any session. |
| `feedback-audience.md`| Audience reactions, comments, engagement signals          | You, after publishing content                         |
| `feedback-system.md`  | Your corrections and preferences when working with agents | Agents log it whenever you give them feedback         |
| `decisions.md`        | Strategic choices and why you made them                   | You or any agent when a decision is made              |

## The Loop

Memory is the engine of improvement. Here's how it works:

1. You create something (content, proposal, decision brief)
2. Gatekeeper reviews it and logs what it learned to `learning-log.md`
3. Over time, patterns emerge
4. Strong patterns get promoted to `C-core/voice-dna.md` or `C-core/project-brief.md`
5. Now every agent knows that pattern permanently

**That's The Loop.** Memory feeds Core. Core feeds agents. Agents create better output. Better output creates better memory.

## The Archive

When files get long (50+ entries), move older entries to `archive/`. Keep the active files lean. Your agents read these files every session, so shorter = faster = cheaper.

Example: move entries older than 30 days to `archive/learning-log-2024-01.md`.

## Tips

- **Be specific.** "The hook worked" is useless. "Opening with a personal failure story got 3x more engagement than opening with a statistic" is gold.
- **Log immediately.** Don't wait. The insight fades fast.
- **Promote aggressively.** If the same pattern appears 3 times, it's a rule. Move it to C-core.
