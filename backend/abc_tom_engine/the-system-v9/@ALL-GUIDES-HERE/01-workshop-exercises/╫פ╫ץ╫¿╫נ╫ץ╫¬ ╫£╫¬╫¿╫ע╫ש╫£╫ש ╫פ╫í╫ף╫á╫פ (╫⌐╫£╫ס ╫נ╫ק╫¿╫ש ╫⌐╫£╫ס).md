4. ודאו שאתם רואים את כל התיקיות (A-agents, B-brain, C-core, T-tools וכו') בסרגל הצד ב-Obsidian. הן בתוך תיקיית the-system שחילצתם מה-ZIP.

### שלב 1: בחרו את המסלול שלכם

לפני שמתחילים, בחרו את המסלול שמתאים לעבודה שלכם:

**תוכן** (Content)
תוכן לפרסום שאנשים קוראים: פוסטים, ניוזלטרים, מאמרים לבלוג.

בשבילכם אם אתם משווקים, יוצרי תוכן, יזמים, כל מי שמפרסם תוכן.

**תקשורת** (Communication)
טקסט מקצועי לנמען ספציפי: מיילים, הודעות סלאק, עדכונים פנימיים, קהילה, חינוך.

בשבילכם אם אתם יזמים, פרילנסרים, יועצים, אנשי HR, מנהלים, כל מי שכותב הודעות מקצועיות.

**חשיבה** (Thinking)
תדריכי החלטות: אתם מביאים החלטה שצריך לקבל, כמה סוכנים מתווכחים עליה מזוויות שונות, ואתם מקבלים תדריך מובנה עם נקודות מבט שונות.

בשבילכם אם אתם מנכ"לים, מנהלי מוצר, בעלי עסקים, מנהלים, כל מי שמקבל החלטות.

**לא בטוחים מה לבחור?** שאלו את עצמכם:
- אתם מפרסמים דברים שאנשים קוראים? כנראה **תוכן**
- אתם כותבים תקשורת מקצועית? כנראה **תקשורת**
- אתם צריכים עזרה בקבלת החלטה? כנראה **חשיבה**

## שלב ב׳: הריצו את הצוות החדש שלכם

*עכשיו עברו ל-Claude Code. הריצו פרומפט אחד בכל פעם. חכו שיסיים לפני שמדביקים את הבא.*

כל 4 הפרומפטים נמצאים בתיקיית המסלול שלכם בתוך `T-tools/01-prompts/`:
- מסלול תוכן ← `T-tools/01-prompts/01-content/`
- מסלול תקשורת ← `T-tools/01-prompts/02-communication/`
- מסלול חשיבה ← `T-tools/01-prompts/03-thinking/`

**פרומפט 1. ספרו לי על עצמכם**

- העתיקו את כל התוכן של `01-learn-from-context.md`
- הדביקו ב-Claude Code ושלחו את קלודי לעשות את שלו.
- Claude קורא את התשובות שלכם ומעדכן אוטומטית את קבצי ה-C-core: `project-brief`, `icp-profile`, `voice-dna`.

**פרומפט 2. לימוד סגנון הכתיבה שלי**

- העתיקו את `02-learn-from-samples.md` והדביקו.
- Claude קורא את 3 הדוגמאות ומעדכן את קבצי ה-Core, האייג'נטים, וה-Memory.

**פרומפט 3. יצירת הפלט שלכם**

- העתיקו את קובץ ה-`03-...` מתיקיית המסלול שלכם והדביקו.
- מלאו את הפרטים:
  - **תוכן:** הנושא או הרעיון לפוסט/ניוזלטר/מאמר
  - **תקשורת:** למי אתם כותבים ועל מה
  - **חשיבה:** ההחלטה שצריך לקבל והאפשרויות שאתם שוקלים
- אדם (סמנכ"ל התפעול שלכם) מנהל את כל הצוות כדי ליצור את הפלט. התוצאה נשמרת ב-`O-output/`.

מה כל מסלול מייצר:

- **תוכן** תוכן לפרסום (פוסט, מאמר, ניוזלטר). 3 סוכנים עובדים יחד: חוקר מוצא זוויות → קופירייטר כותב בקול שלכם → גייטקיפר בודק איכות.
- **תקשורת** טקסט מקצועי לנמען (מייל, הודעת סלאק, עדכון פנימי, קהילה). 3 סוכנים עובדים יחד: אסטרטג מנתח את המצב → קופירייטר כותב בקול שלכם → גייטקיפר בודק איכות.
- **חשיבה** תדריך החלטות. אתם מביאים החלטה שצריך לקבל, ו-4 סוכנים מתווכחים עליה מזוויות שונות (אסטרטג → עורך דין של השטן → ראש מטה → גייטקיפר). אתם מקבלים תדריך מובנה עם נקודות מבט שונות והמלצות.

**פרומפט 4. סגירת ה-Loop**

- העתיקו את `04-close-the-loop.md` והדביקו.
- אדם בודק מה יצרנו, מתעד מה עבד ומה לא ב-M-memory, ומקדם תובנות חזקות ל-C-core.
- זה מה שגורם למערכת להשתפר עם הזמן.
