# Prompt: Create a New Workflow

Use this to build a multi-step workflow that connects your agents.

Copy and paste this into your chat (fill in your workflow idea first):

---

```
I want to create a new workflow for my system. Here's what I need:

---
Workflow name: [e.g., "onboarding-workflow", "review-workflow", "weekly-report-workflow"]
What it does: [Describe the end-to-end process. What goes in? What comes out?]
Which agents should be involved: [e.g., "Researcher, Copywriter, Gatekeeper" or "let Adam decide"]
Trigger: [When should this workflow run? Manual? Weekly? After a specific event?]
---

Please do the following:

1. First, study the existing workflows to understand the structure and style:
   - Read T-tools/03-workflows/content-workflow.md
   - Read T-tools/03-workflows/communication-workflow.md
   - Notice the format: process overview, step-by-step with agent roles, deliverables, checklists

2. Read the brand core so the workflow respects the brand:
   - C-core/project-brief.md
   - C-core/voice-dna.md

3. Check which agents and skills already exist:
   - Scan A-agents/ for available agents
   - Scan T-tools/02-skills/ for available skills

4. Create the new workflow file in T-tools/03-workflows/:
   - Use the same structure as existing workflows
   - Include a process overview diagram (ASCII)
   - Define each step clearly: which agent does what, what they read, what they produce
   - Add deliverable templates for each step
   - Include a checklist at the end
   - End with the signature: > **© Tom Even**
> Workshops & future dates: [www.getagents.today](https://www.getagents.today)
> Newsletter: [www.agentsandme.com](https://www.agentsandme.com)

5. Update any agents that should know about this workflow:
   - If Adam should orchestrate it, note it in the workflow
   - If specific agents have new responsibilities, flag it

6. Update M-memory/learning-log.md:
   - Add an entry documenting the new workflow
   - Note which agents are involved and what triggers it

Show me the new workflow and confirm all updates are done.
```
