# Prompt: Close The Loop

You just sent a professional communication. An email, a proposal, an internal update, an HR message, a community post, a follow-up. Whatever the type, it taught the system something. Now let's capture it. This is The Loop. It's what makes week 10 better than week 1.

Copy and paste this into your chat:

---

```
I just created a professional communication. Now I want to close The Loop so the system learns and improves.

Adam, please manage this:

1. Read what we just created:
   - Check the latest folder in O-output/ (the one we just made)
   - Read the strategist analysis, copywriter draft, gatekeeper review, and final message
   - Note the communication type (proposal / email / update / HR / community / outreach / follow-up)

2. Read current memory and core files:
   - M-memory/learning-log.md
   - M-memory/feedback-audience.md
   - M-memory/feedback-system.md
   - M-memory/decisions.md
   - C-core/project-brief.md
   - C-core/voice-dna.md
   - C-core/icp-profile.md

3. Log what we learned to M-memory/learning-log.md:
   - What worked well in this communication? (tone, structure, opening move, CTA)
   - What felt off or needed revision?
   - Any patterns in how I communicate with this type of recipient?
   - What did the Gatekeeper flag? Any phrases that almost slipped through?
   - Did the approach match the type? (proposal-tone for a proposal, casual-tone for community, etc.)

4. Check if anything is ready to promote to C-core:

   **From learning-log to voice-dna:**
   - Any tone or phrasing that worked across multiple communications? Add it as a rule.
   - Any pattern to avoid? Add it as a "never do this."
   - A signature opening or sign-off worth standardizing? Promote it.

   **From learning-log to icp-profile:**
   - Did we learn something new about how this recipient (or type of recipient) responds?
   - Any insight about what they care about that should inform future messages?

   **From learning-log to decisions.md:**
   - Did this communication set a precedent? (a pricing line, a positioning line, a "we don't do this" line)
   - Any rule that should govern future communications of this type?

5. Sort B-brain/_INBOX/:
   - Move any unsorted items to the right subfolder
   - Delete anything no longer relevant
   - Summarize what you sorted

6. Show me a summary:
   - What we learned from this communication (and what type it was)
   - What got promoted to C-core (if anything)
   - What stayed in Memory (not ready yet)
   - What was sorted from INBOX
   - One specific thing to try differently next time, scoped to this communication type

This is The Loop. Every time you close it, the system gets smarter at the kind of communication you just sent.
```

---

## Why The Loop Matters

Most AI systems are static. You set them up and they stay the same forever.

The ABC-TOM Loop is different. Every message you send teaches the system something. The Loop captures those lessons and bakes them into the system permanently.

**Week 1:** Memory is empty. Messages are rough.
**Week 4:** Memory has patterns. Messages improve.
**Week 8:** Strong patterns get promoted to Core. Messages sound like you.
**Week 12:** The system runs itself. You just steer.

The Loop is the difference between "using AI" and "building an AI team."

---

## When to Use This

Run this prompt after:
- Sending a client proposal
- Sending a sales or outreach email
- Posting a community message
- Delivering an internal update
- Handling an HR or sensitive conversation
- Any communication where the recipient's response matters

Skip it for: drafts you didn't send, throwaway notes, internal scratchpads.
